# Security And Sensitive Content

OneMind is an open research project. It must not become a place where private
coordination problems, personal disputes, sensitive data, or non-consensual
examples are published.

## Do Not Submit

- Private messages, emails, chats, calendars, notes, recordings, or personal
  records.
- Sensitive personal information, including health, finance, legal, workplace,
  family, relationship, immigration, education, or location details.
- Identifying details about real people, organizations, conflicts, negotiations,
  or disputes.
- Examples where someone involved has not consented to public disclosure.
- Credentials, tokens, private keys, API keys, or other secrets.

## Safe Alternatives

- Use synthetic examples.
- Abstract real inspiration until no person, organization, relationship, place,
  or private circumstance can be identified.
- Discuss the structure of a coordination problem without sharing the original
  facts.
- Ask whether an example is safe before posting details.

## Reporting

If you find sensitive content, a secret, or a safety issue in this repository,
do not open a public issue with the details. Email `shivam@shivamk3r.com` with a
short description and the affected location.

If sensitive content appears publicly, maintainers should remove it from the
current tree and assess whether git history cleanup is needed before any public
release.
